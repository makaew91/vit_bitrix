# Модуль управления расписанием врача (vit.schedule)

## Обзор

Модуль для 1С-Битрикс, реализующий управление рабочим расписанием врача через административный интерфейс и публичный компонент для отображения доступных слотов.

## Архитектура

**Подход:** своя таблица + D7 ORM DataManager.

Выбран вместо инфоблоков/HL-блоков, потому что:
- 7 фиксированных записей (дни недели) — инфоблоки избыточны
- Демонстрирует владение D7 DataManager
- Чистый install/uninstall
- Максимально прозрачная структура

**Формат решения:** модуль (`local/modules/vit.schedule/`) + компонент (`local/components/vit/schedule.slots/`).

## База данных

### Таблица `vit_schedule`

| Поле         | Тип         | Описание             |
|--------------|-------------|----------------------|
| ID           | int, PK, AI | Автоинкремент        |
| DAY_OF_WEEK  | int (1–7)   | 1=Пн, 7=Вс          |
| IS_WORKING   | char(1)     | Y/N                  |
| TIME_FROM    | varchar(5)  | HH:MM, начало работы |
| TIME_TO      | varchar(5)  | HH:MM, конец работы  |

Длительность слота хранится в настройках модуля через `COption` (ключ `slot_duration`, значение в минутах).

### Начальные данные (при установке)

Пн–Пт: IS_WORKING=Y, 09:00–18:00
Сб–Вс: IS_WORKING=N, пустое время

Длительность слота по умолчанию: 20 минут.

## Админ-интерфейс

### Страница расписания (`/bitrix/admin/vit_schedule_settings.php`)

Таблица из 7 строк (дни недели):

| День         | Рабочий | Начало | Конец |
|--------------|---------|--------|-------|
| Понедельник  | ☑       | 09:00  | 18:00 |
| Вторник      | ☑       | 09:00  | 18:00 |
| Среда        | ☐       | —      | —     |
| ...          |         |        |       |

- Чекбокс «Рабочий день» — при снятии поля времени дизейблятся
- Время — `<input type="time">`
- Валидация: TIME_FROM < TIME_TO, оба обязательны если день рабочий
- Стандартная Битрикс-форма через `CAdminTabControl`
- Пункт меню в левом сайдбаре: «Расписание врача» (раздел «Сервисы»)

### Настройки модуля (`options.php`)

Длительность слота — выпадающий список: 15 / 20 / 30 / 45 / 60 мин.

## Публичный компонент `vit:schedule.slots`

### Параметры

- `DATE` — дата в формате Y-m-d (по умолчанию сегодня)

### Логика (SlotGenerator)

1. Определить день недели для переданной даты
2. Получить запись из `vit_schedule` для этого дня
3. Если IS_WORKING = N → пустой массив, шаблон показывает «Выходной»
4. Нарезать TIME_FROM–TIME_TO на слоты по длительности из настроек модуля
5. Вернуть массив `["09:00", "09:20", "09:40", ...]`

### Шаблон

- Навигация по дням недели (Пн–Вс с датами текущей недели)
- Сетка кнопок со слотами
- Нерабочие дни — надпись «Выходной»
- Минимальный CSS, без JS-фреймворков

### Вызов

```php
$APPLICATION->IncludeComponent("vit:schedule.slots", "", [
    "DATE" => date("Y-m-d")
]);
```

## Что НЕ входит в scope

- Запись пациента на конкретный слот (бронирование)
- Авторизация / личный кабинет врача
- Редактирование расписания из публичной части
- Интеграция с существующим модулем vit.doctor
- Специализации, типы приёма, услуги, филиалы
